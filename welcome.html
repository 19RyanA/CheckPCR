A word on security

If you are using the extension, you can ignore all of this. Otherwise, please read the following notice.

Because of the way browsers work, webpages cannot access data from other domains unless under very special conditions. This is because it would be dangerous for a malicious page to be able to read data from and send requests to websites such as your bank's while you are logged in (which is very bad for obvious reasons).

This is why Check PCR started off as a chrome extension. Chrome extensions can obtain special permissions to do things such as read and modify data on your current tab or read and modify data on specific websites.
However, normal websites behave the way stated above. The only solution currently out there is to proxy data through the server hosting the online version (which means PCR's data is coming from the same domain as the online version). This means that in order to proxy data from PCR thought the server, your credentials must be passed through it.

So, by submitting your credentials to this site, you are trusting me with the job of not stealing them. I can promise you a thousand times that I have no desire to go against Harker's values and steal your credentials, but it's your call.

If you are concerned about this, you can alternatively use the Chrome extension, which communicates directly with PCR instead of sending data through this server (you can review the source code to make sure of this). Because I chose not to publish it to the Chrome web store, it cannot update itself, so it can become a little annoying to redownload it when there are updates.
However, it has all of the same features of the online version, so don't feel bad about using it instead.
